PolicyShift

A research benchmark for teaching tool-using agents to follow enterprise policies that change over time — without clinging to stale rules.

See how it works Repository

The problem

Real organizations rewrite procedures: materials receiving, lab access, AI/data governance. An agent must apply the policy that was valid at the event time, adapt when versions update, keep still-valid skills, and refuse unsafe shortcuts.

Retrieval alone is not enough — an agent can fetch the right document and still act on an outdated clause. PolicyShift builds a fully synthetic, executable environment so we can compare post-training methods under matched evaluation, with deterministic verifiers instead of hand-wavy LLM-as-judge scores.

Three domains × three versions (1.0 → 1.1 → 2.0) × 120+ tool-use cases. Methods under study: baseline, version-aware RAG, distillation, DPO, continual replay, TeacherBudget selection, and verifier-guided RL (CPU smoke paths by default; optional GPU LoRA documented).
